# 入门介绍

了解 yflow 的核心概念和基本工作流程。

## 核心概念

### 项目 (Project)

项目是组织翻译的基本单元。每个项目代表一个独立的应用或模块：

- **多项目支持**：可以创建多个项目分别管理不同应用的翻译
- **独立配置**：每个项目有独立的语言配置和翻译内容
- **权限管理**：支持为不同用户分配项目级别的权限

### 语言 (Language)

语言定义了你的应用需要支持的所有 locale：

- **默认语言**：设定主语言，其他语言与之对比翻译
- **语言代码**：使用标准 IETF 语言标签（如 `en`, `zh-CN`, `ja-JP`）
- **自动检测**：系统支持自动检测和添加新语言

### 翻译键 (Translation Key)

翻译键是翻译内容的唯一标识：

- **命名规范**：建议使用点分隔符（如 `user.login.title`）
- **层级结构**：支持嵌套组织便于管理
- **描述说明**：可以为每个键添加描述帮助理解用途

### 翻译值 (Translation Value)

翻译值是键在不同语言下的实际文本：

- **矩阵视图**：以表格形式展示所有翻译
- **批量操作**：支持批量导入导出
- **版本历史**：保留翻译修改历史

## 工作流程

### 1. 创建项目

在管理界面创建新项目，配置基本信息：

- 项目名称
- 描述说明
- 默认语言

### 2. 配置语言

为项目添加需要支持的语言：

- 系统预设语言
- 自定义语言
- 设置默认语言

### 3. 添加翻译

通过以下方式添加翻译：

- **手动添加**：在翻译矩阵中直接输入
- **导入**：从 JSON/CSV 文件批量导入
- **CLI 同步**：使用 CLI 工具从代码同步

### 4. 团队协作

邀请团队成员参与翻译工作：

- 分配项目权限
- 使用邀请码注册
- 按角色分工协作

### 5. 集成到应用

使用 CLI 工具同步翻译到本地：

```bash
# 同步翻译到本地
yflow sync

# 在应用中使用
import messages from './locales/en.json'
```

## 数据模型

```
Project (项目)
├── Languages (语言列表)
│   ├── en (英语 - 默认)
│   ├── zh-CN (简体中文)
│   └── ja-JP (日语)
└── Translations (翻译)
    ├── key: "greeting"
    │   ├── en: "Hello"
    │   ├── zh-CN: "你好"
    │   └── ja-JP: "こんにちは"
    └── key: "user.name"
        ├── en: "Name"
        ├── zh-CN: "姓名"
        └── ja-JP: "名前"
```
