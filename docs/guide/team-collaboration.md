# 团队协作

学习如何在 yflow 中管理团队成员和协作翻译。

## 用户角色

### 系统级角色

| 角色 | 权限 |
|------|------|
| **管理员 (Admin)** | 管理系统所有设置、用户管理、语言管理 |
| **成员 (Member)** | 使用平台功能，不能管理系统设置 |

### 项目级角色

| 角色 | 权限 |
|------|------|
| **所有者 (Owner)** | 完全控制项目、管理成员、删除项目 |
| **编辑者 (Editor)** | 编辑翻译、导入导出 |
| **查看者 (Viewer)** | 只读权限 |

## 邀请新成员

### 生成邀请码

只有管理员可以生成邀请码：

1. 登录管理后台
2. 进入「邀请管理」页面
3. 点击「生成邀请码」
4. 设置邀请码有效期和次数限制
5. 分享邀请码给新成员

### 注册新账户

新用户使用邀请码注册：

1. 访问注册页面
2. 输入邀请码
3. 填写账户信息
4. 完成注册

## 管理项目成员

### 添加成员

1. 进入项目详情
2. 点击「成员管理」
3. 输入用户邮箱或用户名搜索
4. 选择角色并添加

### 分配角色

| 角色 | 适用场景 |
|------|----------|
| 所有者 | 项目负责人、技术主管 |
| 编辑者 | 翻译人员、内容编辑 |
| 查看者 | 产品经理、测试人员 |

### 移除成员

在成员管理页面移除成员，移除后该成员将失去项目访问权限。

## 协作工作流

### 推荐的团队结构

```
项目 A
├── @owner: 技术主管
│   ├── @editor: 翻译团队 Lead
│   │   ├── @editor: 翻译人员 A
│   │   └── @editor: 翻译人员 B
│   └── @viewer: 产品经理
```

### 翻译流程

1. **准备阶段**：Owner 创建项目、配置语言
2. **初始导入**：Editor 使用 CLI 导入现有翻译
3. **协作翻译**：团队成员在矩阵中编辑翻译
4. **质量检查**：Viewer 审核并反馈
5. **同步发布**：Editor 使用 CLI 同步到代码库

## 最佳实践

### 权限管理

- 最小权限原则：只授予完成工作所需的最小权限
- 定期审查：定期检查项目成员列表
- 及时移除：成员离开项目后立即移除

### 沟通协作

- 使用翻译描述说明上下文
- 对不确定的翻译添加注释
- 定期同步翻译到代码库
